Aqui você pode perceber a importância de escrever a mensagem na hora do commit, pois assim é possível saber exatamente sobre o que se trata cada commit, inclusive o da primeira linha que foi comitado na aula anterior como exemplo. Boas descrições são essênciais.

Existem informações importantes em cada um deles, como quem fez o commit (já que é possível ter mais de uma pessoa trabalhando no mesmo código), email, quando e inclusive o identificador destacado em amarelo, uma hash única para cada commit, importante para nosso controle de versionamento que depois falaremos.

Podemos utilizar também " git log --author='Nome' " para pegar todos os commits de um determinado programador.

Perceba que o nome do autor fica destacado em vermelho para facilitar a identificação.

Digitando 'git shortlog', você será capaz de ver os autores, quantos commits eles fizeram e quais foram pelas mensagens.

@User
MINGW64
~/Git Directory
(master)
  • $ git shortlog
  • Lucas Rodrigues (3):
    • First version of github platform learning
    • Atualização do arquivo iniciandoRepositorio.hmtl
    • Nova página e atualização de estilo
@User
MINGW64
~/Git Directory
(master)

Por isso é importante colocar boas mensagens. Se tivesse mais contribuintes, eles apareceriam nessa lista

Há outro shortlog mais simples, mostrando apenas o nome e quantidade de commits, só é necessário adicionar -sn ao final do shortlog.

@User
MINGW64
~/Git Directory
(master)
  • $ git shortlog -sn
    • 3 Lucas Rodrigues
@User
MINGW64

Se houvesse outros autores, eles também apareceriam nessa lista.

Podemos também consultar um commit específico para ver o que foi adicionado e removido. Você precisará copiar a hash do commit que você quer analisar. Logo após isso, digite o comando 'git show [hash copiada aqui]'
